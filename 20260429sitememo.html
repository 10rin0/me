CMS/SSG

Publii ：静的サイトジェネレーター（SSG）。公式にはStatic CMS。 Hugoがこう……黒い画面で……文字がいっぱい……に抵抗感があるならおすすめです。

イメージとしては、アプリでダウンロードして、ローカルで作成したものをアップロードするWordPress。

テーマやプラグインの数だけならHugoやWPが断然有利ですが、静的ページを作成したい＋直感的に操作したいのどっちもやりたいのでPubliiにしてみました。

日本語の翻訳もあります。ありがとうございます！🎈

ブロックエディタ、マークダウンエディタ、WYSIWYGエディタが選べるのもよかった。 マークダウンよりWYSIWYGが好きなので…… 投稿ではHTMLコードも直接編集できるので、ルビやクラスを追加したりもできる。 あんまり使うかはわからないけども……。

Google Fontsから好きなフォントも選べるし、 選択できない日本語フォントは、カスタムHTML・CSS機能で直接追加しました。

サイトまるごとの設定でrobots.txtが、 ページごとにnoindex,nofollow等も設定可能でした。

robots.txtで検索サイトへの表示を許可した場合でも、 ChatGPTのボットだけ拒否する細かい設定も可能です。 自分で書くこともできるみたいです。今回はざっくり指定しかしていません。

User-agent: *
Disallow: /

手元でビルドしてFTPでアップロードする方法もあり、 最悪何もわからんでも、手打ちHTMLができて レンタルサーバーにアップロードできるなら、 何とかできる安心感が…初心者には安心……。

ブログとしてならfeed.xmlも出せます。出さないこともできます。 こういうかなり細かい設定が、ボタンをポチポチしてできるのがいいな……と思いました。

投稿（post）と固定ページの作成ができます。 今は固定ページとして作成したものをホームページに設定しています。 WPと同じ感じ。結構ウェブサイトっぽくなるのではないでしょうか！？

メニューで投稿一覧へのリンクができるはずなのですが、 バグか、そもそも出力されていないか？で、うまくいかず、 著者リンクで代用しています。 自分がどこかで設定したのかも…。見逃しているので、有識者、求ム……( ；∀；)

テーマ

Mono ：シンプルでいい感じかなと選びました。公式チームが出している無料テーマ。

メインカラーは数色選べますが、自分で指定可能です。 #9B8EC7を設定しました。

フォントはサイト設定から設定できるものの他、カスタムCSSからも設定しています。 メイリオのウェイトがちょっと軽めなので見づらさを感じるかも… 公式でフォント追加方法もあるので、やってみようと思います。

カスタムCSSの設定

:root {
  --font-heading: 'Orbitron', 'メイリオ', Meiryo, sans-serif;
  --font-entry: 'メイリオ', Meiryo, sans-serif;
  --font-menu: var(--font-entry);
}

カスタムHTMLを使ってGoogle Fonts から直接引っ張ってきてもいいと思うけどな……

プラグイン

Feather Icons ：monoテーマでサポートしているアイコン。 メニュー部分がアイコンになるのがかわいくてお気に入り。

Syntax Highlighter ：Prism.jsが使えるようになります。 投稿でコードエディタが使えるのですが、その部分に反映されます。 コードの色が変わると……嬉しい……。

ホスティング

Github pagesにしました。 https://github.com/10rin0/me

Publii 公式のドキュメントに説明があります。

画像付きなので、かなりわかりやすく、書いてある通りにすればほとんどすべての設定が出来ます。 更新が2年前なので多少異なる部分もありますが、文言はそのままなので、 単語を拾えばたどり着きます。誤差。

私は自分のドメインがあるので設定しましたが、 Github pagesならドメインを持っていなくてもページが公開できます。

＠GitHub Pages/カスタム ドメインを構成する/カスタム ドメインを管理する」の項目で 説明されている通りにするといけました。

CloudFlareでも公開できるらしいのですが、 どっちにしろGithubを仲介する必要があるようなので、 それならGithub pagesでいいか……と思っています。

CloudFlareを使用した場合は、Githubリポジトリはプライベートでいいみたいです。

ドメイン

  • レジストラ：スタードメイン
  • ネームサーバ：CloudFlare

元々ネットオウルのドメインで取得しました。 他サイトとの兼ね合い等も考えて、CloudFlareを使うことにしてみました。 まだ移管はしてないですが、検討中です。

おわりに
